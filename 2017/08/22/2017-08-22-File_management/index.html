设计师应该如何优雅的管理文件 - File Management


之前文章中讲到动效设计中的标注规范，其中有一部分聊到动效文件的管理，这里展开聊一下设计师比较合理的文件管理技巧。

先看一下反例，可能有些时候设计师的项目文件真的会是这样子，当然是个极端反例。

之前在滴滴的时候做一些运营设计需求我是这样管理文件的，按照时间新建一个文件夹，这样看上去很整齐，但是这样管理是很不科学的，首先如果需要找之前某次需求的设计源文件是很难找到的，并不能记住某一天做的是哪个版本需求。

我没有用什么文件管理的工具，都是手动管理，包括目录、版本和命名。基本是根据产品的版本需求来新建目录。所以，就有了下面这个文件目录结构模板，这个是放在我的项目根目录，每次开完需求会我就会直接复制一个EX文件夹重命名为版本名称，然后按照这个文件夹目录模板开始放设计文件及素材文档。不过对于一些日常迭代的需求，则会做适当删减。

总体结构：

  • docs ：需求文档、文案、产品数据、竞品分析、邮件等
  • mockup ：视觉源文件
  • assets ：切图
  • measurement ：标注文件
  • output ：设计效果图
  • material：参考素材文件
  • motion：动效文件


docs ：这里的文档基本是可以扎堆随便放，需求文档、文案、竞品分析、对本次需求有帮助的文档什么都可以放

mockup ：这里放源文件，其实命名为Source更合理，自己能看明白就可以，也可以根据源文件不同再进行分类（比如一些psd智能模板或者模板源文件之类），不过文件数量比较少的情况可以不用再细分目录。

assets ：切图文件可以根据所做的产品不同目录结构进行细分目录

这里要提一下sketch的Symbols，可以按照项目切图所在文件夹直接规范命名，切图会直接导出在相应的文件内。


measurement ：标注文件，这里可以放标注文件，这个事要发给研发宝宝的，所以文件命名和顺序最好和output里的设计文件效果图一致。

output ：这里就是导出的UI效果图。可以根据不同模块细分目录

material：参考素材文件，在看完需求文档后设计师回去找各种素材，全部丢这里，设计的时候就可以直接查看，不建议开太多的网页，这里我有时候也会分两类，一类是和本次需求相关的参考文件，另一类就是在找素材的时候偶遇的很棒的设计师作品，在做完项目需求之后可以再整理这些自己觉得值得参考的设计作品。
注意：此文件夹自己留着看就好，如果自己做的设计文件和参考太类似可能会被说是抄袭，所以妥善保管（不过还是要记住，不会参考优秀作品的设计师不是好美工）

motion：这里我是做完界面之后把需要做动画的页面单独拉到这个文件里，根据不同页面动画细分下一级文件夹。每一个动画目录下又有mockup、assets、mesuerment、output等文件。之前写的这篇文章有专门讲动效设计中的标注规范

总结：

这里讲的是我个人在工作中比较顺手的文件管理方法，大家在不同的设计工作中可以借鉴并总结适合自己工作性质的文件管理习惯。

最后要说一下的是，作为一名设计师，应该有这样一种心态：所有经由我手产出的东西，都必须是被我精心设计过的——不仅仅是最终的视觉稿或标注，我觉的这是设计师的必备技能。在整个工作流程中，对文件的管理都应该做好接收众人审视的准备。让自己的工作更有逻辑性，更高效，也让拿到你文件的同事或者客户一秒看懂你的文件结构找到他们需要的东西，易于修改和补充，并且露出“哇！看起来好专业”。
